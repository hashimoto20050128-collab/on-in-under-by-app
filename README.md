# My Room English ｜ お部屋えいご

小学校 外国語科のデジタル教材。背景の部屋に物を自由に配置し、その配置を**英語だけ**で説明して、聞き手が正しい配置を当てる活動を、クラス全員でリアルタイムに行えます。

- **先生**：進行を操作し、画面をモニター投影
- **児童**：各自の端末で配置・予想・提出
- 同期は **Firebase Realtime Database**。配信は **GitHub Pages** などの静的ホスティングでOK。

---

## 1. ファイル構成
```
classroom-en/
├ index.html          入口（先生／児童を選ぶ）
├ teacher.html        先生用（進行・投影）
├ student.html        児童用（各自の端末）
├ common.js           配置エンジン・採点ロジック
├ styles.css          デザイン
├ firebase-config.js  ★ここに自分のFirebase設定を入れる
├ app-config.js       ★講師の利用コードを設定
└ assets/             背景と物の画像（bg / lamp / pencil / ball / book / plant）
```

## 2. Firebase の準備（5分）
1. [Firebase コンソール](https://console.firebase.google.com/) で新規プロジェクトを作成。
2. 左メニュー **構築 → Realtime Database** を作成（ロケーションは任意、テストモードで開始）。
3. **プロジェクトの設定 → マイアプリ → ウェブアプリ（</>）** を追加し、表示される `firebaseConfig` の値をコピー。
4. `firebase-config.js` の上部を、その値で書きかえる（特に `databaseURL` を忘れずに）。

### データベースのルール（授業期間中の簡易設定）
テスト用に全許可で動きます。ただし誰でも読み書きできるため、**授業期間だけ**にするか、下記のように簡単なクラスコード制限を推奨します。
```json
{
  "rules": {
    "sessions": {
      "$code": { ".read": true, ".write": true }
    }
  }
}
```
> 本番運用や長期公開では Firebase Authentication を併用してください。

## 2.5 利用コード・ルーム番号・モード
- **講師の利用コード**：先生がルームを発行するには `app-config.js` の `TEACHER_CODES` のコードが必要です（例 `"sensei-2026"`）。配布相手に応じて自由に変更・追加できます。※簡易的な制限で、ソースを見れば分かる程度の強さです。
- **ルーム番号**：先生がモードを選んで発行すると、重複しない4桁のルーム番号が自動で割り当てられます。児童はこの番号で参加します。
- **モード**：
  - *クラス全体モード* … 1人が説明し、クラス全員が予想。上位3名（出席番号）を表示。
  - *ペア・グループモード* … グループ内で交代しながら相手の部屋を説明し合い、**一致率（％）** を表示。「つぎの人へ」で交代します（先生はスタート／終了のみ）。
    - グループの作り方は先生が選べます：**ランダム（自動）**／**出席番号順（自動）**／**児童が相手を選ぶ（手動）**。
    - 自動は人数を選んで割り振り、奇数などで端数が出る場合は **2人ペア＋3人組** のように自動で混在させます（1人だけのグループは作りません）。
    - 手動は、児童がペア・グループ内の「いちばん小さい出席番号」を全員で入力して組みます（席順で決まっているペアに便利）。
- **終了**：先生が「終了」を押すと児童はホームに戻り、そのルーム番号（データ）は削除されます。

## 3. 公開する（GitHub Pages の例）
1. このフォルダを GitHub リポジトリにアップロード。
2. リポジトリの **Settings → Pages** で、ブランチ（main / ルート）を選んで公開。
3. 発行された URL（例 `https://ユーザー名.github.io/リポジトリ名/`）を児童に共有。
   - そのまま `index.html` が開きます。ローカルで試す場合は `python3 -m http.server` などでサーバ経由で開いてください（`file://` 直開きはカメラ/保存等が一部制限されます）。

## 4. 授業の流れ（操作手順）
| 流れ | 先生の操作 | 児童の画面 |
|---|---|---|
| ① 説明 | `index.html`→先生→クラスコード入力（例 `6A`）。説明画面を投影 | 待機 |
| ② 配置 | 「配置タイムを開始」。必要なら「物の大きさ変更を許可」ON | コード＋出席番号で参加 → 部屋を作る →（保存）→ **提出** |
| ③ 出題者選び | 「出題スタート（出題者をえらぶ）」で出席番号をランダム表示 | — |
| ④⑤ 説明 | 投影に出題者番号が大きく出る | 出題者：自分の部屋を見ながら英語で説明。他の児童：聞いて予想配置 |
| ⑥ 予想 | 提出人数を見守る | 予想して **提出** |
| ⑦ 結果 | 「結果を発表」→ 正解の部屋・一番近い人・順位を表示 | 正解と自分の得点を確認 |
| ⑧ 繰り返し | 「次のラウンド」で別の出題者へ（③へ戻る） | — |

- 出席番号は「提出済み」かつ「未出題」の児童から自動で選ばれます。全員終わると再抽選の確認が出ます。
- 「配置タイムにもどる」で②からやり直せます。

## 5. 採点の考え方
各物について、正解と予想の**位置の近さ**と**大きさの近さ**を 0〜1 で評価し、全物の平均を **0〜100点**に換算します。
- 位置が近いほど高得点（約0.35以上離れると0点）。位置が主、大きさは従（25%）で、場所が合っているほど大きさの差も効きます。
- 大きさ変更がOFFのときは全員同じサイズなので、位置だけの採点と同じになります。
- 正解で置いていない物を予想でも置かなければ正解扱い、片方だけ置くと0点

しきい値は `common.js` の `_closeness` 内 `TH`（既定 0.35）で調整できます。

## 6. 物品・場面の例（英語表現）
物：lamp / pencil / ball / book / plant / clock
場所：on the table / under the table / in the box / on the wall / on the floor / on the rug
例：*The ball is on the table. / The lamp is under the table. / The clock is on the wall.*

物を増やしたいときは `assets/` に画像を追加し、`common.js` の `OBJECTS` 配列に1行足すだけで反映されます。
